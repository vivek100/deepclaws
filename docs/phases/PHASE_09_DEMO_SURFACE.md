# Phase 9: Demo Surface

## Goal

Build the minimum interface needed to show the live run and the improvement arc.

## Scope

- live run view
- iteration comparison view
- optional Macroscope review story

## Deliverables

- one live-run screen
- one score/diff dashboard
- optional PR-review workflow summary

## Tasks

1. Expose one endpoint or script output suitable for a live run demo.
2. Display baseline vs improved iteration results.
3. Show the prompt or behavior difference between iterations.
4. If GitHub PR flow exists by then, layer in Macroscope review as supporting evidence.

## Validation

- demo can show a single run from question to SQL to result
- demo can show at least one iteration improvement

## Exit criteria

- the team can record or present the project without explaining from raw files alone

## Risks

- spending too long on UI before the benchmark loop is real

## Notes

- The demo should expose real artifacts, not mock progress.
